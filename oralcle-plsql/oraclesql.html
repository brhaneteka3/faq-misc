Oracle SQL FAQ

Databases can be shared-filed based or client-server based

Databases such as Oracle, SQL Server, MySQL (and its offshoot MariaDB) are client-server based.

Microserver Access and File Maker are file based designed for Desktop use.

SQL, PL-SQL and tools used to communicate

PL/SQL is Oracle's SQL implementation which stands for Procedural Language / Structured Query Language.

SQL *PLUS or SQL Developer can be used as a means to communicate to Database Server.

Using Oracle PL/SQL regular expressions

REGEXP_LIKE() Example 1: select prod_name from products where REGEXP_LIKE (prod_name, '1000') order by prod_name; -- that contain 1000

REGEXP_LIKE() Example 2: select prod_name from products where REGEXP_LIKE (prod_name, '.000') order by prod_name; -- can be 1000/2000(. means any character)

REGEXP_LIKE() Example 3: select prod_name from products where REGEXP_LIKE (prod_name, '1000|2000') order by prod_name; -- can be 1000/2000

REGEXP_LIKE() Example 4: select prod_name from products where REGEXP_LIKE (prod_name, '[123] ton') order by prod_name; -- can be 1 or 2 or 3 followed by 'ton'

REGEXP_LIKE() Example 5: select prod_name from products where REGEXP_LIKE (prod_name, '1|2|3 ton') order by prod_name; -- can be 1 or 2 or ('3 ton')

REGEXP_LIKE() Example 6: select prod_name from products where REGEXP_LIKE (prod_name, '[^123] ton') order by prod_name; -- anything but 1 or 2 or 3 followed by ton

REGEXP_LIKE() Example 7: select prod_name from products where REGEXP_LIKE (prod_name, '[1-5] ton') order by prod_name; -- 1 to 5 followed by ton

REGEXP_LIKE() Example 8: select vend_name from vendors where REGEXP_LIKE (vend_name, '\.') order by vend_name; -- dot character

REGEXP_LIKE() Example 9: select vend_name from vendors where REGEXP_LIKE (vend_name, '\\') order by vend_name; -- forward slash character

REGEXP_LIKE() Example 10: select prod_name from products where REGEXP_LIKE (prod_name, '\(\d sticks?\)') order by prod_name; -- brackets followed by a digit followed by stick or sticks

REGEXP_LIKE() Example 11: select prod_name from products where REGEXP_LIKE (prod_name, '\d{4}') order by prod_name; -- a digit * 4

REGEXP_LIKE() Example 12: select prod_name from products where REGEXP_LIKE (prod_name, '[0-9]{4}') order by prod_name; -- a digit * 4

REGEXP_REPLACE() Example 13: With strings as (select 'abc123' s from dual union all select 'abc123' s from dual union all select 'a1b2c3' s from dual) select s "STRING", regexp_replace(s, '[0-9]','') "MODIFIED_STRING" from strings; -- replaces numbers in strings by emptystring

REGEXP_REPLACE() Example 14: With strings as (select 'abc123' s from dual union all select 'abc123' s from dual union all select 'a1b2c3' s from dual) select s "STRING", regexp_replace(s, '[0-9]','',1,1) "MODIFIED_STRING" from strings; -- replaces first number occurrence in strings by emptystring

REGEXP_REPLACE() Example 15: With strings as (select 'abc123' s from dual union all select 'abc123' s from dual union all select 'a1b2c3' s from dual) select s "STRING", regexp_replace(s, '[0-9]','',1,2) "MODIFIED_STRING" from strings; -- replaces second num occurrence in strings by emptystring

REGEXP_REPLACE() Example 16: With strings as (select 'Hello World' s from dual union all select 'Hello World' s from dual union all select 'Hello, World !' s from dual) select s "STRING", regexp_replace(s, ' {2,}', ' ') "MODIFIED_STRING" from strings; -- Convert multiple spaces into a single space

REGEXP_REPLACE() Example 17: WITH date_strings AS ( SELECT '2015-01-01' d from dual union all SELECT '2000-12-31' d from dual union all SELECT '900-01-01' d from dual ) SELECT d "STRING", regexp_replace(d, '([[:digit:]]+)-([[:digit:]]{2})-([[:digit:]]{2})', '\3.\2.\1') "MODIFIED_STRING" FROM date_strings; -- Convert yyyy-mm-dd date formats to dd.mm.yyyy

REGEXP_SUBSTR() Example 18: with strings as ( select 'ABC123' str from dual union all select 'A1B2C3' str from dual union all select '123ABC' str from dual union all select '1A2B3C' str from dual)select regexp_substr(str, '[0-9]') first_number, /* Returns the first number */ regexp_substr(str, '[0-9].*') first_number_followed_by_all, /* Returns the first number and the rest of the string */ regexp_substr(str, '[A-Z][0-9]') first_letter_with_num_after /* Returns the first letter with a following number */ from strings; -- Convert yyyy-mm-dd date formats to dd.mm.yyyy

REGEXP_INSTR() Example 19: SELECT emailID, REGEXP_INSTR(emailID, '\w+@\w+(\.\w+)+') "IS_A_VALID_EMAIL" FROM regexp_temp; -- Query the table columns and check the specified string is contained in the column value; -- Convert yyyy-mm-dd date formats to dd.mm.yyyy

  6. Oracle vs Ansi join: Oracle Database has two syntaxes for joining tables. The proprietary Oracle method. And the ANSI standard way.
    Oracle syntax joins tables in the where clause. ANSI style has a separate join clause. This tutorial will show both methods.
    The ANSI syntax clearly separates the join and filter clauses.
  7. Cross joins Example 1: select * from toys, bricks; select * from toys cross join bricks;
  8. Inner join Example 1: select * from toys, bricks where toy_id = brick_id; select * from toys inner join bricks on toy_id = brick_id; select * from toys join bricks on toy_colour <> brick_colour;
  9. Outer joins: An outer join returns all the rows from one table along with the matching rows from the other.
    An outer join can either be left or right.
    select * from toys left outer join bricks on toy_id = brick_id; select * from toys, bricks where toy_id = brick_id (+);select * from toys right join bricks on toy_id = brick_id; select * from bricks left join toys on toy_id = brick_id;
